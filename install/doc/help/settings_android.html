Android Settings



Enable USB connexion on Android Device

On the Android device press MENU, select Settings > Applications > Development, then enable USB debugging.

Insert SDCard

On Android devices > 1.5, you must insert a SDCard in the phone (otherwise touch events replay will fail). Be careful : the SDCard must be inserted, but NOT mounted (the content of the SDCard should not appear in your explorer).

ATK Monitor

In order to measure resources consumption on Android device, an Android application named "ATK Monitor" is automatically installed and started by ATK on your device. When ATK Monitor is running, you can see a green square on the top left corner of your Android phone:


If ATK is not running, go to the "Applications" page on your phone and launch ATK Monitor:


If ATK Monitor is not installed, you can find its package under <ATK_install_path>/AndroidTools/ATKMonitor.apk and install it manually on your phone.

Android Debug Bridge

ATK uses ADB (Android Debug Bridge) to communicate with Android phones. The standard adb.exe application is provided with ATK. But some phones are using a specific ADB (an adb.exe is provided with the phone drivers). In that case, the standard adb.exe used by ATK may not recognize the phone, and when you plug your phone it won't be detected by ATK.

In such a case, open the Edit > Configuration menu of ATK Test Manager, select the External tools tab. Then check the "Use specific ADB" checkbox and indicate the path to the specific adb.exe application :


Network Monitor

ATK provides a network monitor module (available for Android rooted phones with busybox only). The requested Urls will be displayed as markers on the graph analyzer and on the real time graph.

To enable it check "Use Network Monitor".


Android devices support

If your Android device is still not detected by ATK when plugged, check that it is currently supported. Please see the section "Configuring a new Android device".
