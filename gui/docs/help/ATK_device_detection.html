ATK Device Detection

If a phone is supported by ATK, it will be automatically detected when plugged with USB cable. A "device detection" button is displayed on ATK Test manager (and also on script recorder, with exactly the same behaviour) :




By default, this button displays the first device detected or "No phone" if no phone is connected. The phone displayed will be the one used to record or run the tests.

If more than one phone is connected, click on the button to make the "Connected devices" panel appear :



The list of the phones detected by ATK is displayed. Click on a phone line to select it (the "device detection" button will display the name of the new selected device). A phone can have different statuses :
  • AVAILABLE
    • , means that the phone is connected and available for tests
  • BUSY
    • , means that the phone is connected but its COM port is already used by another application. If a device is busy, running any test will fail. Find and close the other application using the phone, so that it becomes AVAILABLE for test.
  • FAILED
    • , means that the phone is connected, but the last recording or test run has failed.
  • DISCONNECTED
    • , means that the selected phone has been disconnected (if an unselected phone is disconnected, it will just disappear from the list).
When the selected device status is AVAILABLE, the phone icon screen displayed in the "Device detection" button is blue, otherwise it is red :
